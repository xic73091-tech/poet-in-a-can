# 本地隐私方案 Local Privacy Solution

> 你的数据不离开你的手机。这是底线，不是选项。
> Your data never leaves your phone. This is a bottom line, not an option.

---

## 核心原则 Core Principles

1. **数据不出设备 Data Stays on Device**：所有感知、记忆、情绪数据本地存储和处理 All perception, memory, emotion data stored and processed locally
2. **最小权限 Minimum Permissions**：只申请必要的系统权限 Only request necessary system permissions
3. **用户控制 User Control**：用户可以随时查看、导出、删除所有数据 Users can view, export, delete all data anytime
4. **透明可审计 Transparent & Auditable**：所有数据处理逻辑公开可查 All data processing logic is open and auditable

## 本地存储方案 Local Storage Solution

### 存储架构 Storage Architecture

```
/app-data/
├── /memory/
│   ├── short_term.db      # 短期记忆（内存数据库，不持久化）
│   ├── long_term.db       # 长期记忆（SQLite加密）
│   └── index.db           # 记忆索引（SQLite FTS）
├── /emotion/
│   ├── state.json         # 当前情绪状态
│   └── rhythm.json        # 情绪节律配置
├── /perception/
│   ├── cache/             # 感知缓存（临时，定期清理）
│   └── config.json        # 感知配置
├── /identity/
│   ├── soul.json          # 灵魂底色配置
│   └── voice.json         # 声音三层配置
└── /export/
    └── *.json             # 用户导出的数据
```

### SQLite加密 SQLite Encryption

```python
# 使用SQLCipher进行数据库加密
import sqlcipher3

conn = sqlcipher3.connect('long_term.db')
conn.execute("PRAGMA key=?", (user_key,))
conn.execute("PRAGMA cipher_page_size=4096")
conn.execute("PRAGMA kdf_iter=256000")
```

- **加密算法 Encryption**：AES-256-CBC
- **密钥派生 Key Derivation**：PBKDF2，256000次迭代 256000 iterations
- **密钥来源 Key Source**：用户密码 + 设备唯一标识 User password + device ID
- **密钥存储 Key Storage**：iOS Keychain / Android Keystore

### 存储限制 Storage Limits

| 数据类型 Data Type | 存储上限 Limit | 超出处理 Overflow Handling |
|----------|----------|----------|
| 短期记忆 Short-term | 20条 20 items | 蒸馏或遗忘 Distill or forget |
| 长期记忆 Long-term | 200条 200 items | 遗忘最弱记忆 Forget weakest |
| 感知缓存 Perception Cache | 10MB | 定期清理 Periodic cleanup |
| 诗歌存档 Poem Archive | 500首 500 poems | 提示用户导出 Prompt export |

## 感知数据处理 Perception Data Processing

### 照片处理 Photo Processing

```
用户分享照片
    ↓
特征提取（本地）
├── 主色调（像素聚类）
├── 光线（亮度统计）
├── 场景（分类模型）
├── 人脸（检测模型）
└── EXIF（元数据读取）
    ↓
特征存储（轻量级数据结构）
    ↓
原始照片丢弃（不存储）
```

**隐私保护 Privacy Protection：**
- 照片只在内存中处理，不写入磁盘 Photos processed in memory only, not written to disk
- 特征提取后立即释放原始图像 Release raw image immediately after feature extraction
- 不进行人脸识别（只检测数量）No face recognition (only count detection)
- 不读取照片中的文字（除非用户主动分享截图）No text reading (unless user shares screenshot)

### GPS处理 GPS Processing

```
GPS坐标
    ↓
模糊化处理
├── 精度从10米降到100米
├── 存储的是"区域"而非"坐标"
└── 常去地点用聚类中心代替
    ↓
地点语义化
├── 匹配POI数据库（本地）
├── 生成语义标签（家/公司/咖啡店）
└── 不存储原始坐标
    ↓
语义数据存储
```

**隐私保护 Privacy Protection：**
- 不存储精确坐标 No precise coordinates stored
- 不记录移动轨迹 No movement tracking
- 不上传位置数据 No location data uploaded
- 用户可随时清除位置历史 Users can clear location history anytime

### 声音处理 Audio Processing

```
麦克风输入
    ↓
实时特征提取（本地）
├── 分贝水平
├── 频率分布
├── 情绪推断
└── 不保存原始音频
    ↓
特征使用后丢弃
```

**隐私保护 Privacy Protection：**
- 不录音，只做实时分析 No recording, real-time analysis only
- 原始音频不写入任何存储 Raw audio not written to any storage
- 分析结果不包含语音内容 Analysis results contain no speech content
- 用户可完全关闭麦克风权限 Users can fully disable mic permission

## 数据加密方案 Data Encryption

### 传输加密 Transport Encryption
- 由于数据不出设备，无传输加密需求 No transport encryption needed as data stays on device
- 如果未来有云同步功能，使用TLS 1.3 If cloud sync added later, use TLS 1.3

### 存储加密 Storage Encryption

| 数据类型 Data Type | 加密方式 Encryption | 密钥管理 Key Management |
|----------|----------|----------|
| 长期记忆 Long-term Memory | SQLCipher (AES-256) | Keychain/Keystore |
| 情绪状态 Emotion State | 文件级加密 File-level | Keychain/Keystore |
| 灵魂配置 Soul Config | 文件级加密 File-level | Keychain/Keystore |
| 感知缓存 Perception Cache | 不加密（临时数据）Not encrypted (temp) | — |

### 密钥管理 Key Management

```
用户密码
    ↓
PBKDF2（256000次迭代）
    ↓
设备唯一标识（盐值）
    ↓
派生密钥
    ↓
存储在 Keychain/Keystore
```

## 安全威胁与应对 Security Threats & Countermeasures

| 威胁 Threat | 应对措施 Countermeasure |
|------|----------|
| 设备被盗 Device stolen | 数据加密，需要密码解锁 Encrypted, needs password |
| 应用被逆向 App reverse-engineered | 代码混淆，密钥不硬编码 Code obfuscation, no hardcoded keys |
| 内存数据泄露 Memory data leak | 敏感数据用完即清零 Zero sensitive data after use |
| 备份泄露 Backup leak | 加密备份，密钥不在备份中 Encrypted backup, key not in backup |
| 恶意应用读取 Malicious app read | 沙盒隔离，不共享数据 Sandbox isolation, no data sharing |
