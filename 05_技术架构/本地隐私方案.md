# 本地隐私方案

> 你的数据不离开你的手机。这是底线，不是选项。

---

## 核心原则

1. **数据不出设备**：所有感知、记忆、情绪数据本地存储和处理
2. **最小权限**：只申请必要的系统权限
3. **用户控制**：用户可以随时查看、导出、删除所有数据
4. **透明可审计**：所有数据处理逻辑公开可查

---

## 本地存储方案

### 存储架构

```
/app-data/
├── /memory/
│   ├── short_term.db      # 短期记忆（内存数据库，不持久化）
│   ├── long_term.db       # 长期记忆（SQLite加密）
│   └── index.db           # 记忆索引（SQLite FTS）
├── /emotion/
│   ├── state.json         # 当前情绪状态
│   └── rhythm.json        # 情绪节律配置
├── /perception/
│   ├── cache/             # 感知缓存（临时，定期清理）
│   └── config.json        # 感知配置
├── /identity/
│   ├── soul.json          # 灵魂底色配置
│   └── voice.json         # 声音三层配置
└── /export/
    └── *.json             # 用户导出的数据
```

### SQLite加密

```python
# 使用SQLCipher进行数据库加密
import sqlcipher3

conn = sqlcipher3.connect('long_term.db')
conn.execute("PRAGMA key=?", (user_key,))
conn.execute("PRAGMA cipher_page_size=4096")
conn.execute("PRAGMA kdf_iter=256000")
```

- **加密算法**：AES-256-CBC
- **密钥派生**：PBKDF2，256000次迭代
- **密钥来源**：用户密码 + 设备唯一标识
- **密钥存储**：iOS Keychain / Android Keystore

### 存储限制

| 数据类型 | 存储上限 | 超出处理 |
|----------|----------|----------|
| 短期记忆 | 20条 | 蒸馏或遗忘 |
| 长期记忆 | 200条 | 遗忘最弱记忆 |
| 感知缓存 | 10MB | 定期清理 |
| 诗歌存档 | 500首 | 提示用户导出 |

## 本地推理方案

### 小型语言模型部署

| 平台 | 框架 | 模型格式 | 模型大小 |
|------|------|----------|----------|
| iOS | Core ML | .mlmodel | 50-100MB |
| Android | TensorFlow Lite | .tflite | 50-100MB |
| 跨平台 | ONNX Runtime | .onnx | 50-100MB |

### 模型选择

| 功能 | 模型类型 | 要求 |
|------|----------|------|
| 诗歌生成 | 小型语言模型（1-3B参数） | 能生成连贯中文文本 |
| 情感分析 | 文本分类模型 | 识别10种基本情感 |
| 场景识别 | 图像分类模型 | 识别20种常见场景 |
| 人脸检测 | 人脸检测模型 | 检测人脸数量和位置 |

### 推理优化

| 优化技术 | 效果 | 适用场景 |
|----------|------|----------|
| 量化（INT8） | 模型大小减少75%，速度提升2x | 所有模型 |
| 剪枝 | 模型大小减少30-50% | 诗歌生成模型 |
| 知识蒸馏 | 用大模型训练小模型 | 诗歌生成模型 |
| 算子融合 | 推理速度提升20-30% | 所有模型 |
| 缓存 | 避免重复计算 | 常见输入模式 |

### 推理流程

```
用户输入 / 感知信号
    ↓
输入预处理（本地）
    ↓
模型推理（本地，<3秒）
    ↓
后处理（本地）
    ↓
输出结果
```

### 离线推理能力

| 功能 | 离线状态 | 说明 |
|------|----------|------|
| 诗歌生成 | ✅ 完全可用 | 本地模型推理 |
| 情感分析 | ✅ 完全可用 | 本地模型推理 |
| 场景识别 | ✅ 完全可用 | 本地模型推理 |
| 语音合成 | ⚠️ 部分可用 | 基础TSS可用，高质量TSS需要网络 |
| 天气获取 | ⚠️ 部分可用 | 有缓存可用，实时天气需要网络 |
| POI查询 | ⚠️ 部分可用 | 有缓存可用，新地点需要网络 |

## 感知数据处理

### 照片处理

```
用户分享照片
    ↓
特征提取（本地）
├── 主色调（像素聚类）
├── 光线（亮度统计）
├── 场景（分类模型）
├── 人脸（检测模型）
└── EXIF（元数据读取）
    ↓
特征存储（轻量级数据结构）
    ↓
原始照片丢弃（不存储）
```

**隐私保护：**
- 照片只在内存中处理，不写入磁盘
- 特征提取后立即释放原始图像
- 不进行人脸识别（只检测数量）
- 不读取照片中的文字（除非用户主动分享截图）

### GPS处理

```
GPS坐标
    ↓
模糊化处理
├── 精度从10米降到100米
├── 存储的是"区域"而非"坐标"
└── 常去地点用聚类中心代替
    ↓
地点语义化
├── 匹配POI数据库（本地）
├── 生成语义标签（家/公司/咖啡店）
└── 不存储原始坐标
    ↓
语义数据存储
```

**隐私保护：**
- 不存储精确坐标
- 不记录移动轨迹
- 不上传位置数据
- 用户可随时清除位置历史

### 声音处理

```
麦克风输入
    ↓
实时特征提取（本地）
├── 分贝水平
├── 频率分布
├── 情绪推断
└── 不保存原始音频
    ↓
特征使用后丢弃
```

**隐私保护：**
- 不录音，只做实时分析
- 原始音频不写入任何存储
- 分析结果不包含语音内容
- 用户可完全关闭麦克风权限

## 数据加密方案

### 传输加密
- 由于数据不出设备，无传输加密需求
- 如果未来有云同步功能，使用TLS 1.3

### 存储加密

| 数据类型 | 加密方式 | 密钥管理 |
|----------|----------|----------|
| 长期记忆 | SQLCipher (AES-256) | Keychain/Keystore |
| 情绪状态 | 文件级加密 | Keychain/Keystore |
| 灵魂配置 | 文件级加密 | Keychain/Keystore |
| 感知缓存 | 不加密（临时数据） | — |

### 密钥管理

```
用户密码
    ↓
PBKDF2（256000次迭代）
    ↓
设备唯一标识（盐值）
    ↓
派生密钥
    ↓
存储在 Keychain/Keystore
```

## 数据导出与删除

### 数据导出

用户可以导出所有数据，格式为JSON：

```json
{
  "export_date": "2026-05-16",
  "soul_config": { ... },
  "voice_config": { ... },
  "long_term_memories": [ ... ],
  "poems": [ ... ],
  "emotion_history": [ ... ]
}
```

导出文件：
- 不加密（用户自己的数据）
- 可导入到新设备
- 包含完整的人格和记忆

### 数据删除

| 操作 | 效果 | 可逆性 |
|------|------|--------|
| 清除记忆 | 删除所有长期记忆 | 不可逆 |
| 重置人格 | 恢复出厂灵魂底色 | 不可逆 |
| 清除感知缓存 | 删除所有感知历史 | 不可逆 |
| 完全重置 | 删除所有数据，恢复初始状态 | 不可逆 |
| 卸载应用 | 删除所有数据 | 不可逆 |

## 审计与透明

### 数据查看

用户可以查看：
- 当前存储的所有记忆（内容+情感标签+重要度）
- 当前情绪状态和历史
- 感知配置和权限设置
- 存储空间使用情况

### 处理日志

```
[2026-05-16 08:30] 感知：照片分析，场景=户外，人脸=0
[2026-05-16 08:30] 记忆：存入短期记忆，情感=平静，强度=0.4
[2026-05-16 08:31] 情绪：情绪值 5→6，触发=晴天
[2026-05-16 08:31] 表达：生成诗歌，4行，风格=明快
```

日志特性：
- 本地存储，不上传
- 用户可随时查看
- 7天后自动清理
- 不包含敏感内容

## 权限管理

### 必要权限

| 权限 | 用途 | 可关闭 |
|------|------|--------|
| 相机 | 用户主动分享照片时使用 | ✅ |
| 相册 | 用户主动分享照片时使用 | ✅ |
| 位置 | GPS感知 | ✅ |
| 陀螺仪 | 姿态检测和摇晃交互 | ✅ |
| 通知 | 主动开口时发送通知 | ✅ |

### 可选权限

| 权限 | 用途 | 默认 |
|------|------|------|
| 麦克风 | 环境声音感知 | 关闭 |
| 环境光 | 光线感知 | 关闭 |
| 气压计 | 天气辅助推断 | 关闭 |

### 权限关闭影响

每个权限关闭时，诗人会告知用户影响：
- "你关闭了位置权限，我将无法感知你在哪里。"
- "你关闭了相机权限，我将无法看你分享的照片。"

## 安全威胁与应对

| 威胁 | 应对措施 |
|------|----------|
| 设备被盗 | 数据加密，需要密码解锁 |
| 应用被逆向 | 代码混淆，密钥不硬编码 |
| 内存数据泄露 | 敏感数据用完即清零 |
| 备份泄露 | 加密备份，密钥不在备份中 |
| 恶意应用读取 | 沙盒隔离，不共享数据 |

---

> 隐私不是功能，是承诺。用户把生活分享给诗人，诗人有责任保护这些数据。数据不出设备，是这个承诺的技术实现。
