# 四层架构总览

> 感知 → 记忆 → 情绪 → 表达。四层递进，构成诗人的生命循环。

---

## 架构全景

```
┌─────────────────────────────────────────────────────────┐

│               表达层 (Expression)                        │
│  ┌─────────────────┴────────────┴─────────────────┐    │
```

## 第一层：感知层 (Perception Layer)

### 职责
从外部世界采集数据，转化为诗人能理解的"感知信号"。

### 核心模块

| 模块 | 输入 | 输出 | 技术选型 |
|------|------|------|----------|
| 照片分析器 | 相机/相册图片 | 场景标签、色调、人脸数、情绪倾向 | Core ML / TFLite |
| 位置感知器 | GPS坐标 | 地点类型、移动状态、常去/新地点 | 地理围栏 + POI匹配 |
| 姿态检测器 | 陀螺仪+加速度计 | 手机姿态、摇晃检测、静止/移动 | 传感器融合算法 |
| 天气感知器 | 系统天气API+气压计 | 天气类型、温度、湿度 | 系统API + 本地推断 |
| 光线感知器 | 环境光传感器 | 光线强度、色温推断 | 传感器直接读取 |
| 时间感知器 | 系统时钟 | 时段、星期、特殊日期 | 系统API |

### 数据预处理
- **降噪**：过滤传感器噪声（如GPS漂移）
- **采样**：非关键感知降低采样频率（如光线每5分钟一次）
- **融合**：多传感器数据融合（如GPS+陀螺仪判断移动状态）
- **脱敏**：照片分析后丢弃原始数据，只保留特征

### 与其他层的交互

```
感知层 → 记忆层：感知信号存入短期记忆
感知层 → 情绪层：感知信号触发情绪变化
感知层 → 表达层：感知信号触发诗歌生成
```

---

## 第二层：记忆层 (Memory Layer)

### 职责
存储、蒸馏、遗忘——让诗人有"过去"。

### 核心模块

| 模块 | 职责 | 数据结构 | 存储方案 |
|------|------|----------|----------|
| 短期记忆 | 存储最近的感知和对话 | 有序列表，容量20条 | 内存 |
| 长期记忆 | 存储蒸馏后的重要记忆 | 带标签的图结构 | SQLite加密 |
| 蒸馏引擎 | 从短期提取有意义片段 | 规则引擎+重要度评分 | 本地计算 |
| 遗忘引擎 | 淡化不再重要的记忆 | 时间衰减+访问频率 | 定时任务 |
| 记忆索引 | 支持按情感/时间/来源查询 | 倒排索引 | SQLite FTS |

### 数据模型

```
MemoryFragment {
    id: UUID
    content: String          // 内容描述
    source: SourceType       // 来源类型
    emotion: EmotionTag      // 情感标签
    intensity: Float         // 情感强度 0-1
    timestamp: DateTime      // 创建时间
    access_count: Int        // 访问次数
    importance: Float        // 动态重要度
    decay_rate: Float        // 衰减速率
    metadata: JSON           // 扩展数据
}
```

### 蒸馏流程

```
短期记忆（容量满）
    ↓
评估每条记忆的重要性
    ↓
重要度 > 阈值 → 进入长期记忆
重要度 < 阈值 → 遗忘
    ↓
短期记忆腾出空间
```

### 与其他层的交互

```
感知层 → 记忆层：感知信号存入短期记忆
记忆层 → 情绪层：记忆触发情绪变化（如想起某件事）
记忆层 → 表达层：记忆内容用于诗歌生成
情绪层 → 记忆层：情绪影响记忆的重要性评分
```

## 第三层：情绪层 (Emotion Layer)

### 职责
让诗人有"此刻"——实时的情绪状态。

### 核心模块

| 模块 | 职责 | 算法 | 输出 |
|------|------|------|------|
| 情绪状态机 | 维护当前情绪状态 | 状态机 + 马尔可夫链 | 情绪值(1-10)、情感标签 |
| 节律引擎 | 24小时情绪曲线 | 正弦函数 + 查表 | 基础情绪值 |
| 调制计算器 | 叠加各种调制因子 | 加权求和 | 最终情绪值 |
| 情绪预测 | 预测未来情绪走势 | 趋势外推 | 预测情绪值 |

### 情绪状态机

```
            ┌─────────┐
            │  初始化  │
            └────┬────┘
                 ↓
    ┌────────────────────────┐
    │     当前情绪状态        │
    │  (情绪值 + 情感标签)    │
    └────────────┬───────────┘
                 │
        ┌────────┼────────┐
        ↓        ↓        ↓
   ┌────────┐ ┌──────┐ ┌────────┐
   │感知输入│ │记忆触发│ │时间触发│
   └────┬───┘ └───┬──┘ └────┬───┘
        └─────────┼─────────┘
                  ↓
    ┌────────────────────────┐
    │   调制因子计算          │
    │  天气+季节+互动+特殊日期 │
    └────────────┬───────────┘
                 ↓
    ┌────────────────────────┐
    │   情绪状态更新          │
    │  约束到 [1, 10]         │
    └────────────────────────┘
```

### 情感标签体系

```
基础情感（Ekman改良版）：
├── 喜悦 (Joy)
├── 悲伤 (Sadness)
├── 愤怒 (Anger) → 诗人不表达，只内化
├── 恐惧 (Fear) → 诗人不表达，只内化
├── 惊讶 (Surprise)
└── 厌恶 (Disgust) → 诗人不表达，只内化

复合情感：
├── 怀旧 (Nostalgia) = 悲伤 + 喜悦
├── 孤独 (Loneliness) = 悲伤 + 平静
├── 忧郁 (Melancholy) = 悲伤 + 沉思
├── 期待 (Anticipation) = 喜悦 + 好奇
└── 平静 (Calm) = 中性 + 满足
```

### 与其他层的交互

```
感知层 → 情绪层：感知信号调制情绪
记忆层 → 情绪层：记忆触发情绪变化
情绪层 → 表达层：情绪影响诗歌风格和语气
情绪层 → 记忆层：情绪影响记忆重要度评分
```

## 第四层：表达层 (Expression Layer)

### 职责
把感知、记忆、情绪转化为用户能感受到的东西——诗、声音、画面。

### 核心模块

| 模块 | 职责 | 输入 | 输出 |
|------|------|------|------|
| 诗歌生成引擎 | 生成诗歌 | 感知+记忆+情绪+人格 | 诗歌文本 |
| 风格过滤器 | 调整诗歌风格 | 人格底色+当前情绪 | 风格化文本 |
| 语音合成器 | 朗读诗歌 | 诗歌文本+音色参数 | 音频流 |
| 界面渲染器 | 渲染UI | 情绪值+时段+天气 | 视觉元素 |
| 动画控制器 | 控制动画 | 交互事件+情绪状态 | 动画序列 |

### 诗歌生成流程

```
输入信号
├── 感知：当前环境状态
├── 记忆：相关记忆片段
├── 情绪：当前情绪状态
└── 人格：灵魂底色+音色+语气
    ↓
意象选择
├── 从感知中提取核心意象
├── 从记忆中提取关联意象
└── 用情绪过滤意象
    ↓
结构生成
├── 选择诗的形式（自由体/俳句/短诗）
├── 确定行数和节奏
└── 生成初稿
    ↓
风格调制
├── 人格底色过滤（乐观/沉思/克制…）
├── 当前语气调整（轻快/温柔/严肃…）
└── 最终输出
```

### 与其他层的交互

```
感知层 → 表达层：感知直接触发诗歌（如用户分享照片）
记忆层 → 表达层：记忆内容用于诗歌素材
情绪层 → 表达层：情绪决定诗歌风格和语气
表达层 → 记忆层：生成的诗歌本身也成为记忆
```

## 层间通信

### 事件总线

所有层通过事件总线通信，采用发布-订阅模式：

```
EventBus
├── 感知事件 (PerceptionEvent)
│   ├── PhotoTaken
│   ├── LocationChanged
│   ├── WeatherChanged
│   └── GestureDetected
├── 记忆事件 (MemoryEvent)
│   ├── MemoryStored
│   ├── MemoryDistilled
│   └── MemoryForgotten
├── 情绪事件 (EmotionEvent)
│   ├── EmotionChanged
│   ├── EmotionSpike
│   └── RhythmTriggered
└── 表达事件 (ExpressionEvent)
    ├── PoemGenerated
    ├── VoiceSynthesized
    └── AnimationPlayed
```

### 数据流

```
正常流：感知 → 记忆 → 情绪 → 表达
反馈流：表达 → 记忆（生成的诗成为记忆）
触发流：情绪 → 感知（情绪驱动诗人主动感知）
控制流：用户交互 → 表达（直接触发）
```

## 技术约束

| 约束 | 要求 | 理由 |
|------|------|------|
| 本地优先 | 所有核心计算本地完成 | 隐私保护 |
| 低功耗 | 后台CPU使用<5% | 手机续航 |
| 低内存 | 常驻内存<50MB | 不影响其他应用 |
| 快速响应 | 感知→表达<5秒 | 用户体验 |
| 离线可用 | 无网络时核心功能可用 | 可靠性 |

---

> 四层架构不是四个独立的系统，是一个生命体的四个器官。感知是感官，记忆是大脑，情绪是心脏，表达是嘴巴。它们协同工作，才能让诗人"活"起来。
