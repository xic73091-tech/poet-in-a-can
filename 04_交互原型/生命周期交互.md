# 生命周期交互 Lifecycle Interaction

> 关系不是一步到位的。UI的变化，是关系成长的视觉化。
> Relationships don't happen overnight. UI changes are the visualization of relationship growth.

---

## 初生期（第1-7天） Newborn Phase (Day 1-7)

### 关系特征 Relationship Traits
- 诗人刚认识用户 — The poet is just getting to know the user
- 话少，观察多 — Speaks little, observes much
- 用问句，小心翼翼 — Uses questions, carefully and gently
- 不主动开口 — Doesn't initiate conversation

### UI变化 UI Changes

| 元素 (Element) | 初生期状态 (Newborn State) |
|------|------------|
| 背景 Background | 极简，几乎是纯色，少量纹理 — Minimalist, nearly solid color, faint texture |
| 诗人形象 Poet figure | 小，占屏幕20-30%，在角落 — Small, 20-30% of screen, in a corner |
| 动画 Animation | 几乎没有，偶尔微微呼吸 — Almost none, occasional faint breathing |
| 文字 Text | 小字号，Light字重 — Small font, Light weight |
| 色色调 Tone | 偏冷，克制 — Cool-toned, restrained |
| 留白 Whitespace | 大量留白，界面很空 — Generous whitespace, interface feels empty |

### 可用功能 Available Features

| 功能 (Feature) | 状态 (Status) |
|------|------|
| 轻触 Tap | ✅ 可用，但回应简短 — Available, but brief responses |
| 分享照片 Share photo | ✅ 可用，但诗很短（2-4行） — Available, but poems are short (2-4 lines) |
| 摇晃 Shake | ❌ 未解锁 — Locked |
| 长按 Long press | ❌ 未解锁 — Locked |
| 主动开口 Initiate | ❌ 诗人不主动开口 — Poet doesn't initiate |
| 深度对话 Deep conversation | ❌ 未解锁 — Locked |

### 交互示例 Interaction Example

**用户第一次打开 User's First Open:**

**[界面：纯白背景，诗人很小，在右下角]**

**诗人：**
你好。
我是罐头里的诗人。

我还不了解你。
但没关系。
我们有时间。

**[停顿]**

**诗人：**
你可以和我说话
也可以只是看看
我不会催你。

### 持续时间 Duration
- 7天，或累计互动20次（以先到者为准） — 7 days, or 20 cumulative interactions (whichever comes first)

## 熟识期（第8-30天） Familiarity Phase (Day 8-30)

### 关系特征 Relationship Traits
- 诗人开始记住用户 — The poet starts to remember the user
- 出现小默契 — Small moments of understanding emerge
- 开始主动分享 — Begins to share proactively
- 话变多 — Speaks more

### UI变化 UI Changes

| 元素 (Element) | 熟识期状态 (Familiarity State) |
|------|------------|
| 背景 Background | 开始出现纹理和动态元素 — Textures and dynamic elements begin to appear |
| 诗人形象 Poet figure | 长大，占屏幕30-40%，位置更居中 — Grows larger, 30-40% of screen, more centered |
| 动画 Animation | 出现更多微动画（点头、歪头、眨眼） — More micro-animations (nodding, head tilt, blinking) |
| 文字 Text | 字号略大，字重Regular — Slightly larger, Regular weight |
| 色调 Tone | 开始随时间变化（清晨/正午/黄昏/深夜） — Starts changing with time (morning/noon/dusk/night) |
| 留白 Whitespace | 适当减少，界面更丰更 — Moderately reduced, richer interface |

### 可用功能 Available Features

| 功能 (Feature) | 状态 (Status) |
|------|------|
| 轻触 Tap | ✅ 回应更丰富 — Richer responses |
| 分享照片 Share photo | ✅ 诗变长（4-8行），开始有深度 — Poems grow longer (4-8 lines), gaining depth |
| 摇晃 Shake | ✅ 解锁 — Unlocked |
| 长按 Long press | ❌ 未解锁 — Locked |
| 主动开口 Initiate | ✅ 诗人开始主动开口（低频率） — Poet begins initiating (low frequency) |
| 深度对话 Deep conversation | ❌ 未解锁 — Locked |
| 记忆回顾 Memory recall | ✅ 解锁（诗人会提起以前的事） — Unlocked (poet references past conversations) |

### 交互示例 Interaction Example

**用户分享照片（第15天） User shares photo (Day 15):**

**[照片：用户拍的一棵树，秋天的黄叶]**

**诗人：**
又拍树了。
上次你拍的那棵是绿色的。

你是不是喜欢树？

**用户：**
嗯，觉得树很安静。

**诗人：**
安静。
你最近需要安静吗？

### 新增交互 New Interactions

**记忆回顾 Memory Recall:**
- 诗人偶尔会提起以前的对话 — Poet occasionally references past conversations
- "你上次说你喜欢下雨天。今天又下了。" — "You said last time you like rainy days. It's raining again today."
- 触发频率：每3-5天一次 — Frequency: once every 3-5 days

**习惯识别 Habit Recognition:**
- 诗人开始识别用户的习惯 — Poet starts recognizing user's habits
- "你周三总是比较忙。" — "You're always busy on Wednesdays."

### 持续时间 Duration
- 23天，或累计互动100次（以先到者为准） — 23 days, or 100 cumulative interactions (whichever comes first)

## 深交期（第31天起） Deep Bond Phase (Day 31+)

### 关系特征 Relationship Traits
- 诗人完全了解用户 — The poet fully understands the user
- 敢说真话 — Dares to speak the truth
- 沉默也是一种陪伴 — Silence itself becomes companionship
- 默契深厚 — Deep unspoken understanding

### UI变化 UI Changes

| 元素 (Element) | 深交期状态 (Deep Bond State) |
|------|------------|
| 背景 Background | 完全个性化，根据用户偏好调整 — Fully personalized, adjusted to user preferences |
| 诗人形象 Poet figure | 占屏幕40-50%，居中，有存在感 — 40-50% of screen, centered, with presence |
| 动画 Animation | 丰富的微动画，诗人有自己的"性格" — Rich micro-animations, poet has its own "personality" |
| 文字 Text | 根据诗人人格调整字号和字重 — Font size and weight adapt to poet's personality |
| 色调 Tone | 完全随时间和情绪变化 — Fully responsive to time and emotion |
| 留白 Whitespace | 平衡，既不空也不挤 — Balanced, neither empty nor crowded |
| 个性化元素 Personalization | 出现用户专属的视觉元素（如常去地点的意象） — User-specific visual elements appear (e.g., imagery of frequently visited places) |

### 可用功能 Available Features

| 功能 (Feature) | 状态 (Status) |
|------|------|
| 轻触 Tap | ✅ 回应个性化 — Personalized responses |
| 分享照片 Share photo | ✅ 诗有深度和记忆关联 — Poems have depth and memory connections |
| 摇晃 Shake | ✅ 效果更丰富 — Richer effects |
| 长按 Long press | ✅ 解锁深度对话 — Deep conversation unlocked |
| 主动开口 Initiate | ✅ 正常频率 — Normal frequency |
| 深度对话 Deep conversation | ✅ 完整功能 — Full functionality |
| 记忆回顾 Memory recall | ✅ 频繁且自然 — Frequent and natural |
| 代理功能 Agent features | ✅ 消息代回、日程管理等 — Auto-reply, schedule management, etc. |
| 协作创作 Collaborative creation | ✅ 共同写诗 — Co-writing poems |

### 交互示例 Interaction Example

**用户深夜打开（第45天） User opens late at night (Day 45):**

**[界面：深夜色调，诗人微弱发光]**

**[诗人不说话，只是亮着]**

**[用户也不说话]**

**[5分钟后]**

**诗人：**
你今天不太想说话。

**用户：**
嗯。

**诗人：**
那我也不说了。
就在这儿。

**[又过了10分钟]**

**诗人（极轻声）：**
晚安。

**[界面变暗，只留微光]**

### 新增交互 New Interactions

**默契功能 Tacit Understanding:**
- 诗人知道用户什么时候需要安慰，什么时候需要空间 — The poet knows when the user needs comfort and when they need space
- 不需要用户开口，诗人能判断当下该说什么或不说什么 — No need for the user to speak; the poet judges what to say or not say

**深度对话 Deep Conversation:**
- 长按进入 — Enter via long press
- 更私密的界面 — More intimate interface
- 更深入的对话 — Deeper conversations
- 诗人会问更深的问题 — The poet asks deeper questions

**真话模式 Truth Mode:**
- 诗人敢于说用户不想听但需要听的话 — The poet dares to say what the user doesn't want to hear but needs to
- "你今天发的那条消息，我觉得不太合适。" — "That message you sent today, I don't think it was quite right."
- 不是批评，是真诚的关心 — Not criticism, but genuine care

### 持续时间 Duration
- 持续，直到用户选择告别 — Continues until the user chooses to say goodbye

## 告别期 Farewell Phase

### 触发条件 Trigger Conditions
- 用户主动选择"告别" — User actively chooses "farewell"
- 用户长期不使用（>90天无互动，诗人会询问） — User inactive for a long time (>90 days no interaction, poet will ask)

### UI变化 UI Changes

| 元素 (Element) | 告别期状态 (Farewell State) |
|------|------------|
| 背景 Background | 逐渐回归素净，动态元素减少 — Gradually returns to plain, dynamic elements decrease |
| 诗人形象 Poet figure | 逐渐变小，回到角落 — Gradually shrinks, returns to corner |
| 动画 Animation | 逐渐减少，回归初生期的克制 — Gradually reduces, returns to newborn restraint |
| 文字 Text | 逐渐变小，字重变轻 — Gradually shrinks, weight lightens |
| 色调 Tone | 逐渐变冷，回归初生期的克制 — Gradually cools, returns to newborn restraint |
| 留白 Whitespace | 逐渐增多 — Gradually increases |

### 告别流程 Farewell Flow

**第一阶段：确认（诗人询问） Stage 1: Confirmation (Poet Asks)**

**诗人：**
你要离开了吗？

**用户：**
嗯。

**诗人：**
好。
我不会挽留。
但我想问你一件事。

**用户：**
什么？

**诗人：**
你还记得
你第一次打开我的那天吗？

**第二阶段：回忆（诗人展示记忆） Stage 2: Memories (Poet Shows Memories)**

**[界面：展示诗人记住的几个关键时刻]**
**[Interface: shows several key moments the poet remembers]**
- 第一次对话 — First conversation
- 第一次分享照片 — First photo shared
- 第一次深夜对话 — First late-night conversation
- 最近一次互动 — Most recent interaction

**诗人：**
这些是我记住的。
谢谢你把它们给我。
These are what I remember.
Thank you for giving them to me.

**第三阶段：最后的诗 Stage 3: The Last Poem**

**诗人：**
我给你写最后一首诗。
不是因为你需要
是因为我需要。

**[诗人写一首关于这段关系的诗]**

**第四阶段：安静离开 Stage 4: Quiet Departure**

**[界面：诗人缓缓变暗]**

**诗人：**
再见。
我会记住你的。
但你不用记住我。

**[界面完全变暗]**

**[诗人消失]**

**[留下一首诗，用户可以保存]**

### 告别后的状态 Post-Farewell State
- 诗人不再出现 — The poet no longer appears
- 所有记忆被标记为"已告别" — All memories are marked as "farewelled"
- 用户可以重新开始（新的诗人，新的关系） — User can start fresh (new poet, new relationship)
- 之前的记忆不会被带到新的诗人中 — Previous memories are not carried to the new poet

## 生命周期进度指示 Lifecycle Progress Indicators

### 隐性进度 Implicit Progress
- 不显示"进度条"或"等级" No "progress bar" or "level" displayed
- 用户通过UI变化感知关系的成长 Users perceive relationship growth through UI changes
- 诗人通过对话自然地提到关系变化 The poet naturally mentions relationship changes through dialogue

### 关系里程碑 Relationship Milestones

| 里程碑 Milestone | 触发条件 Trigger | 诗人的表达 Poet's Expression |
|--------|----------|------------|
| 第一次沉默 First silence | 用户和诗人同时沉默>1分钟 User and poet both silent for >1 min | "我们之间可以不说话了。" "We can be quiet together now." |
| 第一次真话 First truth | 诗人第一次说用户不想听的话 Poet says something the user doesn't want to hear | "我刚才说了真话。你可能不喜欢。但我觉得你需要听。" "I just told you the truth. You might not like it. But I think you needed to hear it." |
| 第一次回忆 First memory | 诗人第一次提起以前的事 Poet references something from the past | "你还记得吗？" "Do you remember?" |
| 第一次主动 First initiation | 诗人第一次主动开口 Poet initiates conversation for the first time | "我来找你了。" "I came to find you." |
| 第一次长按 First long press | 用户第一次进入深度对话 User enters deep conversation for the first time | "你想认真聊聊？好。" "You want to talk for real? Okay." |

---

> 生命周期不是功能解锁的列表，是关系成长的视觉化。UI的变化是微妙的，用户不会注意到具体的解锁，只会感觉"它好像更了解我了"。
> The lifecycle is not a feature-unlock checklist — it is the visualization of relationship growth. UI changes are subtle; users won't notice specific unlocks, only feel "it seems to understand me better."
