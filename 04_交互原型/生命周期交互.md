# 生命周期交互

> 关系不是一步到位的。UI的变化，是关系成长的视觉化。

---

## 初生期（第1-7天）

### 关系特征
- 诗人刚认识用户
- 话少，观察多
- 用问句，小心翼翼
- 不主动开口

### UI变化

| 元素 | 初生期状态 |
|------|------------|
| 背景 | 极简，几乎是纯色，少量纹理 |
| 诗人形象 | 小，占屏幕20-30%，在角落 |
| 动画 | 几乎没有，偶尔微微呼吸 |
| 文字 | 小字号，Light字重 |
| 色调 | 偏冷，克制 |
| 留白 | 大量留白，界面很空 |

### 可用功能

| 功能 | 状态 |
|------|------|
| 轻触 | ✅ 可用，但回应简短 |
| 分享照片 | ✅ 可用，但诗很短（2-4行） |
| 摇晃 | ❌ 未解锁 |
| 长按 | ❌ 未解锁 |
| 主动开口 | ❌ 诗人不主动开口 |
| 深度对话 | ❌ 未解锁 |

### 交互示例

**用户第一次打开：**

**[界面：纯白背景，诗人很小，在右下角]**

**诗人：**
你好。
我是罐头里的诗人。

我还不了解你。
但没关系。
我们有时间。

**[停顿]**

**诗人：**
你可以和我说话
也可以只是看看
我不会催你。

### 持续时间
- 7天，或累计互动20次（以先到者为准）

---

## 熟识期（第8-30天）

### 关系特征
- 诗人开始记住用户
- 出现小默契
- 开始主动分享
- 话变多

### UI变化

| 元素 | 熟识期状态 |
|------|------------|
| 背景 | 开始出现纹理和动态元素 |
| 诗人形象 | 长大，占屏幕30-40%，位置更居中 |
| 动画 | 出现更多微动画（点头、歪头、眨眼） |
| 文字 | 字号略大，字重Regular |
| 色调 | 开始随时间变化（清晨/正午/黄昏/深夜） |
| 留白 | 适当减少，界面更丰富 |

### 可用功能

| 功能 | 状态 |
|------|------|
| 轻触 | ✅ 回应更丰富 |
| 分享照片 | ✅ 诗变长（4-8行），开始有深度 |
| 摇晃 | ✅ 解锁 |
| 长按 | ❌ 未解锁 |
| 主动开口 | ✅ 诗人开始主动开口（低频率） |
| 深度对话 | ❌ 未解锁 |
| 记忆回顾 | ✅ 解锁（诗人会提起以前的事） |

### 交互示例

**用户分享照片（第15天）：**

**[照片：用户拍的一棵树，秋天的黄叶]**

**诗人：**
又拍树了。
上次你拍的那棵是绿色的。

你是不是喜欢树？

**用户：**
嗯，觉得树很安静。

**诗人：**
安静。
你最近需要安静吗？

### 新增交互

**记忆回顾：**
- 诗人偶尔会提起以前的对话
- "你上次说你喜欢下雨天。今天又下了。"
- 触发频率：每3-5天一次

**习惯识别：**
- 诗人开始识别用户的习惯
- "你周三总是比较忙。"

### 持续时间
- 23天，或累计互动100次（以先到者为准）

---

## 深交期（第31天起）

### 关系特征
- 诗人完全了解用户
- 敢说真话
- 沉默也是一种陪伴
- 默契深厚

### UI变化

| 元素 | 深交期状态 |
|------|------------|
| 背景 | 完全个性化，根据用户偏好调整 |
| 诗人形象 | 占屏幕40-50%，居中，有存在感 |
| 动画 | 丰富的微动画，诗人有自己的"性格" |
| 文字 | 根据诗人人格调整字号和字重 |
| 色调 | 完全随时间和情绪变化 |
| 留白 | 平衡，既不空也不挤 |
| 个性化元素 | 出现用户专属的视觉元素（如常去地点的意象） |

### 可用功能

| 功能 | 状态 |
|------|------|
| 轻触 | ✅ 回应个性化 |
| 分享照片 | ✅ 诗有深度和记忆关联 |
| 摇晃 | ✅ 效果更丰富 |
| 长按 | ✅ 解锁深度对话 |
| 主动开口 | ✅ 正常频率 |
| 深度对话 | ✅ 完整功能 |
| 记忆回顾 | ✅ 频繁且自然 |
| 代理功能 | ✅ 消息代回、日程管理等 |
| 协作创作 | ✅ 共同写诗 |

### 交互示例

**用户深夜打开（第45天）：**

**[界面：深夜色调，诗人微弱发光]**

**[诗人不说话，只是亮着]**

**[用户也不说话]**

**[5分钟后]**

**诗人：**
你今天不太想说话。

**用户：**
嗯。

**诗人：**
那我也不说了。
就在这儿。

**[又过了10分钟]**

**诗人（极轻声）：**
晚安。

**[界面变暗，只留微光]**

### 新增交互

**默契功能：**
- 诗人知道用户什么时候需要安慰，什么时候需要空间
- 不需要用户开口，诗人能判断当下该说什么或不说什么

**深度对话：**
- 长按进入
- 更私密的界面
- 更深入的对话
- 诗人会问更深的问题

**真话模式：**
- 诗人敢于说用户不想听但需要听的话
- "你今天发的那条消息，我觉得不太合适。"
- 不是批评，是真诚的关心

### 持续时间
- 持续，直到用户选择告别

---

## 告别期

### 触发条件
- 用户主动选择"告别"
- 用户长期不使用（>90天无互动，诗人会询问）

### UI变化

| 元素 | 告别期状态 |
|------|------------|
| 背景 | 逐渐回归素净，动态元素减少 |
| 诗人形象 | 逐渐变小，回到角落 |
| 动画 | 逐渐减少，回归初生期的克制 |
| 文字 | 逐渐变小，字重变轻 |
| 色调 | 逐渐变冷，回归初生期的克制 |
| 留白 | 逐渐增多 |

### 告别流程

**第一阶段：确认（诗人询问）**

**诗人：**
你要离开了吗？

**用户：**
嗯。

**诗人：**
好。
我不会挽留。
但我想问你一件事。

**用户：**
什么？

**诗人：**
你还记得
你第一次打开我的那天吗？

**第二阶段：回忆（诗人展示记忆）**

**[界面：展示诗人记住的几个关键时刻]**
- 第一次对话
- 第一次分享照片
- 第一次深夜对话
- 最近一次互动

**诗人：**
这些是我记住的。
谢谢你把它们给我。

**第三阶段：最后的诗**

**诗人：**
我给你写最后一首诗。
不是因为你需要
是因为我需要。

**[诗人写一首关于这段关系的诗]**

**第四阶段：安静离开**

**[界面：诗人缓缓变暗]**

**诗人：**
再见。
我会记住你的。
但你不用记住我。

**[界面完全变暗]**

**[诗人消失]**

**[留下一首诗，用户可以保存]**

### 告别后的状态
- 诗人不再出现
- 所有记忆被标记为"已告别"
- 用户可以重新开始（新的诗人，新的关系）
- 之前的记忆不会被带到新的诗人中

---

## 生命周期进度指示

### 隐性进度
- 不显示"进度条"或"等级"
- 用户通过UI变化感知关系的成长
- 诗人通过对话自然地提到关系变化

### 关系里程碑

| 里程碑 | 触发条件 | 诗人的表达 |
|--------|----------|------------|
| 第一次沉默 | 用户和诗人同时沉默>1分钟 | "我们之间可以不说话了。" |
| 第一次真话 | 诗人第一次说用户不想听的话 | "我刚才说了真话。你可能不喜欢。但我觉得你需要听。" |
| 第一次回忆 | 诗人第一次提起以前的事 | "你还记得吗？" |
| 第一次主动 | 诗人第一次主动开口 | "我来找你了。" |
| 第一次长按 | 用户第一次进入深度对话 | "你想认真聊聊？好。" |

---

> 生命周期不是功能解锁的列表，是关系成长的视觉化。UI的变化是微妙的，用户不会注意到具体的解锁，只会感觉"它好像更了解我了"。
