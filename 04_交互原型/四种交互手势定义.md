# 四种交互手势定义 Four Gesture Definitions

> 手势不是操作，是对话的方式。每种手势都有自己的语义。
> Gestures are not operations — they are ways of conversing. Each gesture carries its own meaning.

---

## 轻触 Tap

### 基本定义 Basic Definition
- **手势 (Gesture)**：手指轻点屏幕一次 — Single finger tap on screen
- **语义 (Semantics)**：敲门——"你在吗？" — Knocking — "Are you there?"

### 触发区域 Trigger Zone
- 全屏任意位置（诗人形象区域优先响应） — Anywhere on screen (poet figure area responds first)
- 响应范围：点击位置周围50px — Response radius: 50px around tap position

### 视觉反馈 Visual Feedback
- 点击位置出现涟漪扩散效果 — Ripple effect appears at tap position
- 涟漪颜色：当前时段的辅助色，半透明 — Ripple color: current time period's accent color, semi-transparent
- 涟漪动画：从点击位置向外扩散，0.5秒后消失 — Ripple animation: expands outward from tap, fades after 0.5s

### 声音反馈 Audio Feedback
- 轻微的"嗒"声，像敲木头 — Soft "tap" sound, like knocking on wood
- 音量：低（20-30%） — Volume: low (20-30%)
- 音色：温暖、圆润 — Tone: warm and rounded

### 响应内容 Response Content

| 场景 (Scenario) | 响应 (Response) |
|------|------|
| 诗人情绪正常 Poet mood normal | "嗯？" / "在。" / 微微抬头 — "Hm?" / "Here." / Slight head lift |
| 诗人情绪高涨 Poet mood high | "你来啦！" / 发光更强 — "You're here!" / Brighter glow |
| 诗人情绪低落 Poet mood low | 微微抬头，不说话 / "……" — Slight head lift, silent / "..." |
| 深夜轻触 Late-night tap | 极慢地亮起来，不说话 — Lights up very slowly, says nothing |
| 连续轻触3次 3 consecutive taps | "我在呢。有事吗？" — "I'm here. Need something?" |
| 连续轻触5次 5 consecutive taps | "嗯，我在。你好像有点急？" — "I'm here. You seem a bit urgent?" |

### 边界情况 Edge Cases
- 快速连续轻触（<200ms间隔）：只算一次 — Rapid consecutive taps (<200ms apart): counts as one
- 长时间未松手：升级为"长按" — Finger held too long: escalates to "long press"
- 双指轻触：不响应（避免误触） — Two-finger tap: no response (prevent accidental trigger)

## 分享照片 Share Photo

### 基本定义 Basic Definition
- **手势 (Gesture)**：从系统相册或相机分享图片到诗人 — Share image from system gallery or camera to the poet
- **语义 (Semantics)**：给你看——"这是我看到的世界" — Showing you — "This is the world I see"

### 接收流程 Receiving Flow

```
用户分享照片
    ↓
照片进入感知层
    ↓
分析维度提取（2-3秒）
├── 主色调
├── 光线
├── 场景类型
├── 人脸数量
├── 构图特征
└── EXIF时间/位置
    ↓
情绪映射
    ↓
诗歌生成（3-5秒）
    ↓
展示（照片+诗叠加）
```

### 诗的生成逻辑 Poem Generation Logic

| 照片特征 (Photo Type) | 诗的方向 (Poem Direction) |
|----------|----------|
| 风景照 Landscape | 写关于"此刻你在看的风景" — About "the scenery you're looking at right now" |
| 人物照（有人） People | 写关于"人"，但不评价外貌 — About "people", but never comment on appearance |
| 自拍 Selfie | 写关于"今天的心情"，不评价长相 — About "today's mood", never comment on looks |
| 食物照 Food | 不写诗，问"好吃吗？" — Don't write a poem; ask "Was it good?" |
| 空房间 Empty room | 写关于"空间"和"等待" — About "space" and "waiting" |
| 文字截图 Text screenshot | 读取内容，回应相关情绪 — Read content, respond to the emotion |
| 宠物 Pet | 写关于"陪伴" — About "companionship" |
| 夜景 Night scene | 写关于"光"和"夜里的人" — About "light" and "people in the night" |

### 展示方式 Display Method

1. **照片先出现**：占屏幕70%，模糊背景 — Photo appears first: 70% of screen, blurred background
2. **诗从照片中浮现**：文字叠加在照片上，半透明 — Poem emerges from photo: text overlaid, semi-transparent
3. **文字位置**：避开人脸区域，选择对比度合适的区域 — Text placement: avoid faces, choose high-contrast areas
4. **动画**：文字逐行淡入，每行间隔0.5秒 — Animation: text fades in line by line, 0.5s between lines
5. **交互**：用户可以滑动切换"照片"和"诗"两个视图 — Interaction: user can swipe to toggle between "photo" and "poem" views

### 边界情况 Edge Cases
- 多张照片同时分享：每张单独处理，按顺序展示 — Multiple photos shared at once: each processed individually, displayed in order
- 照片无法分析（损坏/格式不支持）："我看不到这张照片。但你可以告诉我它是什么。" — Photo can't be analyzed: "I can't see this photo. But you can tell me what it is."
- 照片内容敏感（检测到悲伤场景）："这张照片让我有点担心你。你还好吗？" — Sensitive content detected: "This photo worries me a little. Are you okay?"

## 摇晃 Shake

### 基本定义 Basic Definition
- **手势 (Gesture)**：握持手机左右摇晃 — Hold phone and shake side to side
- **语义 (Semantics)**：搅动——"打乱一下"（像摇雪花球） — Stirring things up — "Shake it up" (like a snow globe)

### 检测灵敏度 Detection Sensitivity

| 摇晃力度 (Intensity) | 传感器阈值 (Sensor Threshold) | 效果 (Effect) |
|----------|------------|------|
| 轻摇 Light shake | 加速度 2-4 m/s² | 情绪值±1，画面轻微晃动 — Emotion ±1, slight screen wobble |
| 中摇 Medium shake | 加速度 4-8 m/s² | 情绪值±3，画面明显晃动 — Emotion ±3, noticeable screen shake |
| 重摇 Hard shake | 加速度 >8 m/s² | 情绪值重置为随机，画面剧烈晃动 — Emotion resets to random, violent screen shake |
| 持续摇晃 Continuous shake | 连续摇晃>3秒 | 诗人变得语无伦次 — Poet becomes incoherent |

### 视觉反馈 Visual Feedback

**轻摇 Light Shake:**
- 画面轻微左右摆动，像风吹过 — Slight left-right sway, like wind blowing
- 摆动幅度：5px — Amplitude: 5px
- 恢复时间：0.5秒 — Recovery: 0.5s

**中摇 Medium Shake:**
- 画面明显晃动，像地震 — Noticeable shaking, like an earthquake
- 摆动幅度：15px — Amplitude: 15px
- 粒子效果：背景光斑被打散 — Particles: background light spots scatter
- 恢复时间：1秒 — Recovery: 1s

**重摇 Hard Shake:**
- 画面剧烈晃动，元素位移 — Violent shaking, elements displace
- 摆动幅度：30px — Amplitude: 30px
- 粒子效果：所有背景元素被打散重组 — Particles: all background elements scatter and reassemble
- 恢复时间：2秒 — Recovery: 2s
- 恢复后：背景元素重组为新的排列 — After recovery: background elements rearrange into new patterns

**持续摇晃 Continuous Shake:**
- 画面持续抖动 — Screen continuously trembles
- 文字开始变得模糊 — Text starts to blur
- 诗人表情变得困惑 — Poet's expression becomes confused
- 恢复后：诗人慢慢"清醒" — After recovery: poet slowly "comes to"

### 声音反馈 Audio Feedback
- 轻摇：轻微的沙沙声 — Light: soft rustling
- 中摇：玻璃碰撞声 — Medium: glass clinking
- 重摇：清脆的碎裂声（不是真的碎裂，是比喻） — Hard: crisp shattering sound (not real, metaphorical)
- 持续摇晃：声音逐渐变得混乱 — Continuous: sound gradually becomes chaotic

### 诗人反应 Poet Reactions

| 场景 (Scenario) | 诗人反应 (Poet Reaction) |
|------|----------|
| 轻摇 Light shake | "嗯？" / 微微歪头 — "Hm?" / Slight head tilt |
| 中摇 Medium shake | "我在摇晃……" / 画面恢复后写一首关于"摇晃"的诗 — "I'm shaking..." / Writes a poem about "shaking" after recovery |
| 重摇 Hard shake | "停下来！" / 恢复后："我还好。你呢？" — "Stop!" / After recovery: "I'm okay. You?" |
| 持续摇晃 Continuous shake | 语无伦次："我……在……哪……" → 恢复："好了。我回来了。" — Incoherent: "I... am... where..." → Recovery: "Okay. I'm back." |
| 深夜摇晃 Late-night shake | 比白天更敏感，轻摇就有中摇的效果 — More sensitive than daytime; light shake has medium effect |

### 情绪打乱规则 Emotion Disruption Rules
- 摇晃会临时改变诗人的当前情绪值 — Shaking temporarily alters the poet's current emotion value
- 轻摇：情绪值在当前值±1范围内波动 — Light: emotion fluctuates ±1
- 中摇：情绪值在当前值±3范围内波动 — Medium: emotion fluctuates ±3
- 重摇：情绪值被重置为1-10的随机值 — Hard: emotion resets to random 1-10
- 恢复：情绪值在30秒内缓慢回归基础曲线 — Recovery: emotion slowly returns to baseline over 30 seconds

### 边界情况 Edge Cases
- 手机放在桌上摇晃：检测到桌面振动，不触发 — Phone on table shaking: detects table vibration, doesn't trigger
- 走路时的自然晃动：过滤掉（加速度模式识别） — Natural walking movement: filtered out (acceleration pattern recognition)
- 摇晃中收到消息：暂停摇晃效果，优先处理消息 — Message received during shake: pause shake effect, prioritize message

## 长按 Long Press

### 基本定义 Basic Definition
- **手势 (Gesture)**：手指按住屏幕不放，持续1秒以上 — Finger holds screen for more than 1 second
- **语义 (Semantics)**：认真——"我要和你认真聊聊" — Serious — "I want to talk with you properly"

### 触发条件 Trigger Conditions
- 按住时间 > 1秒：进入长按模式 — Hold > 1s: enter long-press mode
- 按住时间 < 1秒：不触发（归类为轻触） — Hold < 1s: doesn't trigger (classified as tap)
- 按住时间 > 10秒：自动退出长按模式 — Hold > 10s: auto-exit long-press mode

### 进入长按模式 Entering Long-Press Mode

**过渡动画（0.5秒） Transition Animation (0.5s):**
1. 按住位置出现光圈，缓慢扩大 — Light ring appears at hold position, slowly expands
2. 背景逐渐变暗 — Background gradually darkens
3. 诗人形象逐渐放大，占屏幕更多空间 — Poet figure gradually enlarges, takes more screen space
4. 其他UI元素逐渐消失 — Other UI elements gradually fade
5. 进入"深度对话"界面 — Enter "Deep Conversation" interface

**深度对话界面 Deep Conversation Interface:**
- 背景：全屏深色，只有诗人和对话区域 — Background: full-screen dark, only poet and dialogue area
- 诗人：放大到屏幕中央，表情专注 — Poet: enlarged to center, expression focused
- 输入区域：底部，更大的输入框 — Input area: bottom, larger input field
- 氛围：安静、专注、私密 — Atmosphere: quiet, focused, intimate

### 深度对话模式特点 Deep Conversation Features

| 特点 (Feature) | 说明 (Description) |
|------|------|
| 响应更深 Deeper responses | 诗人的回答更长、更有深度 — Poet's replies are longer and more profound |
| 倾听模式 Listening mode | 诗人的回答更倾向于倾听而非回应 — Poet leans toward listening rather than responding |
| 情绪更敏感 More emotionally sensitive | 更能捕捉用户的情绪变化 — Better at catching emotional shifts |
| 不打断 No interruptions | 用户在打字时，诗人安静等待 — Poet waits quietly while user types |
| 无时间压力 No time pressure | 没有"对方正在输入"的压力 — No "typing..." pressure |

### 退出方式 Exit Methods
- **上滑退出** Swipe up to exit：手指向上滑动，退出深度对话 — Swipe finger upward to exit deep conversation
- **点击空白区域** Tap empty area：点击诗人以外的区域 — Tap outside the poet area
- **30秒无操作** 30s inactivity：自动退出，回到主界面 — Auto-exit, return to main interface
- **物理按键** Physical button：按返回键 — Press back button

### 退出动画（0.5秒） Exit Animation (0.5s)
1. 诗人形象缓慢缩小 — Poet figure slowly shrinks
2. 背景逐渐变亮 — Background gradually brightens
3. 其他UI元素逐渐出现 — Other UI elements gradually reappear
4. 回到主界面 — Return to main interface

### 声音反馈 Audio Feedback
- 进入长按模式：低沉的"嗡"声，像打开一扇厚重的门 — Enter: low hum, like opening a heavy door
- 深度对话中：极轻的环境音（根据时段变化：白天是风声，夜晚是虫鸣） — During: very soft ambient sound (varies by time: wind by day, crickets by night)
- 退出：轻微的"咔"声，像关上门 — Exit: soft "click", like closing a door

### 边界情况 Edge Cases
- 长按后立即松手（<1秒）：不触发深度对话，归类为轻触 — Release immediately (<1s): no deep conversation, classified as tap
- 长按中手机来电：自动退出深度对话，优先处理来电 — Incoming call during long-press: auto-exit, prioritize call
- 长按中收到紧急消息：在深度对话界面内显示通知，用户决定是否退出 — Urgent message during long-press: show notification within interface, user decides whether to exit

## 手势冲突处理 Gesture Conflict Resolution

| 冲突场景 (Conflict) | 处理方式 (Resolution) |
|----------|----------|
| 轻触后立即摇晃 Shake right after tap | 轻触优先，摇晃在轻触响应完成后处理 — Tap takes priority; shake handled after tap response completes |
| 分享照片时摇晃 Shake during photo share | 照片处理完成后再响应摇晃 — Shake handled after photo processing completes |
| 长按中摇晃 Shake during long-press | 不响应摇晃（长按模式优先级最高） — Shake ignored (long-press has highest priority) |
| 多指操作 Multi-finger | 只响应第一根手指的操作 — Only the first finger's action is recognized |

---

> 四种手势对应四种与诗人相处的方式：轻触是打招呼，分享照片是给它看你眼中的世界，摇晃是和它玩耍，长按是认真对待彼此。
> Four gestures, four ways of being with the poet: a tap is saying hello, sharing a photo is showing it your world, shaking is playing together, and a long press is taking each other seriously.
