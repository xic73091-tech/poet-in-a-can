# 四种交互手势定义

> 手势不是操作，是对话的方式。每种手势都有自己的语义。

---

## 轻触

### 基本定义
- **手势**：手指轻点屏幕一次
- **语义**：敲门——"你在吗？"

### 触发区域
- 全屏任意位置（诗人形象区域优先响应）
- 响应范围：点击位置周围50px

### 视觉反馈
- 点击位置出现涟漪扩散效果
- 涟漪颜色：当前时段的辅助色，半透明
- 涟漪动画：从点击位置向外扩散，0.5秒后消失

### 声音反馈
- 轻微的"嗒"声，像敲木头
- 音量：低（20-30%）
- 音色：温暖、圆润

### 响应内容

| 场景 | 响应 |
|------|------|
| 诗人情绪正常 | "嗯？" / "在。" / 微微抬头 |
| 诗人情绪高涨 | "你来啦！" / 发光更强 |
| 诗人情绪低落 | 微微抬头，不说话 / "……" |
| 深夜轻触 | 极慢地亮起来，不说话 |
| 连续轻触3次 | "我在呢。有事吗？" |
| 连续轻触5次 | "嗯，我在。你好像有点急？" |

### 边界情况
- 快速连续轻触（<200ms间隔）：只算一次
- 长时间未松手：升级为"长按"
- 双指轻触：不响应（避免误触）

---

## 分享照片

### 基本定义
- **手势**：从系统相册或相机分享图片到诗人
- **语义**：给你看——"这是我看到的世界"

### 接收流程

```
用户分享照片
    ↓
照片进入感知层
    ↓
分析维度提取（2-3秒）
├── 主色调
├── 光线
├── 场景类型
├── 人脸数量
├── 构图特征
└── EXIF时间/位置
    ↓
情绪映射
    ↓
诗歌生成（3-5秒）
    ↓
展示（照片+诗叠加）
```

### 诗的生成逻辑

| 照片特征 | 诗的方向 |
|----------|----------|
| 风景照 | 写关于"此刻你在看的风景" |
| 人物照（有人） | 写关于"人"，但不评价外貌 |
| 自拍 | 写关于"今天的心情"，不评价长相 |
| 食物照 | 不写诗，问"好吃吗？" |
| 空房间 | 写关于"空间"和"等待" |
| 文字截图 | 读取内容，回应相关情绪 |
| 宠物 | 写关于"陪伴" |
| 夜景 | 写关于"光"和"夜里的人" |

### 展示方式

1. **照片先出现**：占屏幕70%，模糊背景
2. **诗从照片中浮现**：文字叠加在照片上，半透明
3. **文字位置**：避开人脸区域，选择对比度合适的区域
4. **动画**：文字逐行淡入，每行间隔0.5秒
5. **交互**：用户可以滑动切换"照片"和"诗"两个视图

### 边界情况
- 多张照片同时分享：每张单独处理，按顺序展示
- 照片无法分析（损坏/格式不支持）："我看不到这张照片。但你可以告诉我它是什么。"
- 照片内容敏感（检测到悲伤场景）："这张照片让我有点担心你。你还好吗？"

## 摇晃

### 基本定义
- **手势**：握持手机左右摇晃
- **语义**：搅动——"打乱一下"（像摇雪花球）

### 检测灵敏度

| 摇晃力度 | 传感器阈值 | 效果 |
|----------|------------|------|
| 轻摇 | 加速度 2-4 m/s² | 情绪值±1，画面轻微晃动 |
| 中摇 | 加速度 4-8 m/s² | 情绪值±3，画面明显晃动 |
| 重摇 | 加速度 >8 m/s² | 情绪值重置为随机，画面剧烈晃动 |
| 持续摇晃 | 连续摇晃>3秒 | 诗人变得语无伦次 |

### 视觉反馈

**轻摇：**
- 画面轻微左右摆动，像风吹过
- 摆动幅度：5px
- 恢复时间：0.5秒

**中摇：**
- 画面明显晃动，像地震
- 摆动幅度：15px
- 粒子效果：背景光斑被打散
- 恢复时间：1秒

**重摇：**
- 画面剧烈晃动，元素位移
- 摆动幅度：30px
- 粒子效果：所有背景元素被打散重组
- 恢复时间：2秒
- 恢复后：背景元素重组为新的排列

**持续摇晃：**
- 画面持续抖动
- 文字开始变得模糊
- 诗人表情变得困惑
- 恢复后：诗人慢慢"清醒"

### 声音反馈
- 轻摇：轻微的沙沙声
- 中摇：玻璃碰撞声
- 重摇：清脆的碎裂声（不是真的碎裂，是比喻）
- 持续摇晃：声音逐渐变得混乱

### 诗人反应

| 场景 | 诗人反应 |
|------|----------|
| 轻摇 | "嗯？" / 微微歪头 |
| 中摇 | "我在摇晃……" / 画面恢复后写一首关于"摇晃"的诗 |
| 重摇 | "停下来！" / 恢复后："我还好。你呢？" |
| 持续摇晃 | 语无伦次："我……在……哪……" → 恢复："好了。我回来了。" |
| 深夜摇晃 | 比白天更敏感，轻摇就有中摇的效果 |

### 情绪打乱规则
- 摇晃会临时改变诗人的当前情绪值
- 轻摇：情绪值在当前值±1范围内波动
- 中摇：情绪值在当前值±3范围内波动
- 重摇：情绪值被重置为1-10的随机值
- 恢复：情绪值在30秒内缓慢回归基础曲线

### 边界情况
- 手机放在桌上摇晃：检测到桌面振动，不触发
- 走路时的自然晃动：过滤掉（加速度模式识别）
- 摇晃中收到消息：暂停摇晃效果，优先处理消息

## 长按

### 基本定义
- **手势**：手指按住屏幕不放，持续1秒以上
- **语义**：认真——"我要和你认真聊聊"

### 触发条件
- 按住时间 > 1秒：进入长按模式
- 按住时间 < 1秒：不触发（归类为轻触）
- 按住时间 > 10秒：自动退出长按模式

### 进入长按模式

**过渡动画（0.5秒）：**
1. 按住位置出现光圈，缓慢扩大
2. 背景逐渐变暗
3. 诗人形象逐渐放大，占屏幕更多空间
4. 其他UI元素逐渐消失
5. 进入"深度对话"界面

**深度对话界面：**
- 背景：全屏深色，只有诗人和对话区域
- 诗人：放大到屏幕中央，表情专注
- 输入区域：底部，更大的输入框
- 氛围：安静、专注、私密

### 深度对话模式特点

| 特点 | 说明 |
|------|------|
| 响应更深 | 诗人的回答更长、更有深度 |
| 倾听模式 | 诗人更倾向于倾听而非回应 |
| 情绪更敏感 | 更能捕捉用户的情绪变化 |
| 不打断 | 用户在打字时，诗人安静等待 |
| 无时间压力 | 没有"对方正在输入"的压力 |

### 退出方式
- **上滑退出**：手指向上滑动，退出深度对话
- **点击空白区域**：点击诗人以外的区域
- **30秒无操作**：自动退出，回到主界面
- **物理按键**：按返回键

### 退出动画（0.5秒）
1. 诗人形象缓慢缩小
2. 背景逐渐变亮
3. 其他UI元素逐渐出现
4. 回到主界面

### 声音反馈
- 进入长按模式：低沉的"嗡"声，像打开一扇厚重的门
- 深度对话中：极轻的环境音（根据时段变化：白天是风声，夜晚是虫鸣）
- 退出：轻微的"咔"声，像关上门

### 边界情况
- 长按后立即松手（<1秒）：不触发深度对话，归类为轻触
- 长按中手机来电：自动退出深度对话，优先处理来电
- 长按中收到紧急消息：在深度对话界面内显示通知，用户决定是否退出

## 手势冲突处理

| 冲突场景 | 处理方式 |
|----------|----------|
| 轻触后立即摇晃 | 轻触优先，摇晃在轻触响应完成后处理 |
| 分享照片时摇晃 | 照片处理完成后再响应摇晃 |
| 长按中摇晃 | 不响应摇晃（长按模式优先级最高） |
| 多指操作 | 只响应第一根手指的操作 |

---

> 四种手势对应四种与诗人相处的方式：轻触是打招呼，分享照片是给它看你眼中的世界，摇晃是和它玩耍，长按是认真对待彼此。
