# 线下电话亭装置说明

> 一次通话，一生的诗。

---

## 装置概念

一个独立的电话亭，走进去，拿起听筒，和一个AI诗人进行一次通话。通话结束后，你会带走一首只属于你的诗。

这不是产品体验，是一次艺术事件。

---

### 物理设计

| 项目 | 规格 |

| 屏幕 | 嵌入墙内的小屏幕（7寸），显示诗句 |

| 摄像头 | 可选，用于情绪检测（需用户同意） |

## 交互流程

### 完整体验（约5-8分钟）

```
1. 接近（30秒）
   ├── 用户看到电话亭
   ├── 门上有文字："一次通话，一生的诗"
   └── 用户推门进入

2. 进入（30秒）
   ├── 门自动关闭
   ├── 灯光缓慢变亮
   ├── 环境音响起（根据当前时间：白天是风声，夜晚是虫鸣）
   └── 屏幕显示："拿起听筒"

3. 通话开始（30秒）
   ├── 用户拿起听筒
   ├── 听筒里传来诗人的声音
   ├── 诗人："你好。"
   └── 停顿，等待用户回应

4. 对话（3-5分钟）
   ├── 诗人问3-5个问题
   ├── 问题层层递进
   ├── 用户用语音回答
   └── 诗人实时回应

5. 诗的生成（30秒）
   ├── 诗人："我给你写了一首诗。"
   ├── 听筒里传来诗的朗读
   └── 屏幕上同时显示诗句

6. 离开（30秒）
   ├── 打印机吐出一张纸，上面是那首诗
   ├── 用户取走诗
   ├── 屏幕显示："谢谢你的这次通话。"
   └── 灯光缓慢变暗，门解锁

7. 带走
   ├── 用户带走那张纸
   └── 纸上还有二维码，可以回听朗读
```

## 通话内容

### 诗人的问题（由浅入深）

**第一层：表层**
- "你今天怎么来的？"
- "外面天气怎么样？"
- "你最近一次笑是什么时候？"

**第二层：感受**
- "你最近在想什么？"
- "有什么事一直没说出口？"
- "你上次哭是什么时候？"

**第三层：深处**
- "如果给现在的自己写一封信，你会写什么？"
- "你最想对谁说一句话？说什么？"
- "你觉得什么是你一直在等的？"

### 问题选择逻辑

- 根据用户的回答动态调整下一个问题
- 如果用户回避某个问题，不追问，换一个方向
- 如果用户情绪激动，放缓节奏
- 最多问5个问题，不贪多

### 诗的生成逻辑

```
用户的回答
    ↓
关键词提取
    ↓
情感分析
    ↓
意象选择
    ↓
诗歌生成（本地模型，3-5秒）
    ↓
语音合成朗读
    ↓
屏幕显示+打印
```

## 技术实现

### 硬件需求

| 设备 | 型号建议 | 用途 |
|------|----------|------|
| 主机 | Mac Mini / Intel NUC | 运行本地模型 |
| 电话机 | 改装的复古电话机 | 用户交互 |
| 屏幕 | 7寸嵌入式显示屏 | 显示诗句 |
| 打印机 | 热敏打印机 | 打印诗 |
| 音箱 | 嵌入式音箱 | 环境音+朗读 |
| 麦克风 | 嵌入式麦克风 | 采集语音 |
| 灯光 | 可调色温LED灯条 | 氛围灯光 |
| 门锁 | 电磁锁 | 门禁控制 |

### 软件架构

```
┌─────────────────────────────┐
│       交互控制层            │
│  语音识别 → 对话管理 → 语音合成 │
├─────────────────────────────┤
│       诗歌生成层            │
│  意象提取 → 诗歌生成 → 风格调制 │
├─────────────────────────────┤
│       设备控制层            │
│  灯光 → 门锁 → 打印机 → 屏幕  │
├─────────────────────────────┤
│       硬件层                │
│  麦克风 → 音箱 → 电话机 → 主机 │
└─────────────────────────────┘
```

### 网络方案

| 方案 | 说明 | 优缺点 |
|------|------|--------|
| 全本地 | 所有计算本地完成 | 最隐私，但需要高性能硬件 |
| 混合 | 语音识别用云端，诗歌生成本地 | 平衡隐私和性能 |
| 本地优先 | 优先本地，网络不好时降级 | 最实用 |

推荐：**全本地方案**，使用Mac Mini + 本地模型。
