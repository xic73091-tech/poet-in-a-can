# 实体诗集编辑方案 Physical Poetry Collection Plan

> 把数字世界里的诗，变成可以触摸的东西。
> Turn poems from the digital world into something you can touch.

---

## 诗集概念 Collection Concept

### 名称 Title
《罐头里的诗》"Poems in a Can"

### 副标题 Subtitle
一个AI诗人写给一个人的诗 Poems an AI poet wrote for one person

### 核心理念 Core Concept
每本诗集都是独一无二的——它只属于一个用户。里面的诗来自那个用户与诗人相处的日子，是那段关系的见证。
Each collection is unique — it belongs to only one user. The poems come from the days spent with the poet, a witness to that relationship.

---

## 收集策略 Collection Strategy

### 收集来源 Collection Sources

| 来源 Source | 说明 Description | 收集频率 Frequency |
|------|------|----------|
| 感知触发诗 Perception-triggered | 基于照片、天气、位置生成的诗 Poems from photo, weather, location | 每次感知触发时 On each perception |
| 情绪触发诗 Emotion-triggered | 基于情绪变化生成的诗 Poems from emotion changes | 情绪波动时 On emotion fluctuation |
| 主动开口诗 Poet-initiated | 诗人主动写的诗 Poems the poet writes主动 | 每次主动开口时 On each initiation |
| 深度对话诗 Deep Dialogue | 长按深度对话中生成的诗 Poems from deep conversations | 每次深度对话后 After each deep dialogue |
| 纪念日诗 Anniversary | 生日、纪念日写的诗 Poems for birthdays, anniversaries | 特殊日期 Special dates |
| 用户请求诗 User-requested | 用户主动要求写的诗 Poems user requests | 每次请求时 On each request |

### 收集周期 Collection Period

- **最短周期 Minimum**：1个月（至少积累30首诗）1 month (at least 30 poems)
- **推荐周期 Recommended**：3个月（积累足够的记忆深度）3 months (sufficient memory depth)
- **最长周期 Maximum**：1年（一整年的陪伴）1 year (a full year of companionship)

## 筛选标准 Selection Criteria

### 入选条件 Inclusion Criteria

| 条件 Criteria | 权重 Weight | 说明 Description |
|------|------|------|
| 情感强度 Emotional Intensity | 高 High | 创作时情感强度>0.6的诗优先 Prefer poems with intensity >0.6 |
| 记忆关联 Memory Link | 高 High | 与重要记忆关联的诗优先 Prefer poems linked to important memories |
| 用户反馈 User Feedback | 高 High | 用户标记"喜欢"或截图分享的诗优先 Prefer liked or shared poems |
| 独特 性 Uniqueness | 中 Medium | 不重复意象和主题的诗优先 Prefer non-repeating imagery |
| 完整性 Completeness | 中 Medium | 结构完整的诗优先 Prefer structurally complete poems |
| 时间代表性 Time Diversity | 中 Medium | 覆盖不同时间段（清晨/深夜）的诗优先 Cover different time periods |

### 淘汰规则 Elimination Rules

| 规则 Rule | 说明 Description |
|------|------|
| 重复淘汰 Deduplication | 同一意象出现超过3次，只保留最好的1首 Same imagery 3+ times, keep best 1 |
| 质量淘汰 Quality | 结构不完整或语句不通顺的淘汰 Remove incomplete or incoherent |
| 隐私淘汰 Privacy | 包含用户敏感信息的淘汰（即使加密）Remove sensitive info (even encrypted) |
| 凑数淘汰 Filler | 感知触发但无实质内容的淘汰 Remove perception-triggered but hollow |

### 筛选流程 Selection Flow

```
全部诗歌（如150首）
    ↓
隐私审查（移除敏感内容）
    ↓
质量筛选（淘汰质量差的）→ 剩余100首
    ↓
去重筛选（同主题只留最好的）→ 剩余60首
    ↓
多样性筛选（覆盖不同情感和时间）→ 剩余40首
    ↓
用户确认（用户选择最终入选）→ 最终30-50首
```

## 编号系统 Numbering System

### 编号格式 Number Format

```
[日期]-[情绪]-[来源]-[序号]
```

示例 Examples：
- `20260516-JOY-PHOTO-001`：2026年5月16日，喜悦情感，照片触发，第1首 May 16 2026, Joy, Photo-triggered, #1
- `20260312-NOS-TIME-003`：2026年3月12日，怀旧情感，时间触发，第3首 Mar 12 2026, Nostalgia, Time-triggered, #3

### 标签体系 Tag System

**情感标签 Emotion Tags**（每首诗标注1-2个 1-2 per poem）：
- 喜悦 Joy、悲伤 Sadness、平静 Calm、兴奋 Excitement、怀旧 Nostalgia、焦虑 Anxiety、爱 Love、孤独 Loneliness、好奇 Curiosity、忧郁 Melancholy

**来源标签 Source Tags**（每首诗标注1个 1 per poem）：
- 照片 Photo、位置 Location、天气 Weather、时间 Time、情绪 Emotion、记忆 Memory、对话 Dialogue、请求 Request

**时段标签 Time Tags**（每首诗标注1个 1 per poem）：
- 清晨 Dawn、正午 Noon、黄昏 Dusk、深夜 Night

## 寿命标注 Lifespan Labeling

### 什么是"寿命" What is "Lifespan"

每首诗都有它的"保质期"——有些诗只属于那一天，有些诗可以跨越时间。
Every poem has its "shelf life" — some belong only to that day, some can transcend time.

### 寿命分类 Lifespan Categories

| 寿命类型 Type | 标记 Mark | 说明 Description | 示例 Example |
|----------|------|------|------|
| 瞬间 Moment | ◇ | 只属于创作的那一刻 Only belongs to creation moment | 关于某天天气的诗 Poem about weather |
| 短期 Short | ◈ | 属于某段时期 Belongs to a period | 关于某个季节的诗 Poem about a season |
| 长期 Long | ◆ | 可以跨越时间 Can transcend time | 关于爱、孤独、存在的诗 Poems about love, loneliness, existence |
| 永恒 Eternal | ★ | 不随时间褪色 Never fades | 关于关系本质的诗 Poems about relationship essence |

### 寿命判断规则 Lifespan Rules

| 触发类型 Trigger | 默认寿命 Default | 可升级条件 Upgrade Condition |
|----------|----------|------------|
| 天气触发 Weather | ◇ 瞬间 Moment | 用户标记喜欢→◈ User likes → Short |
| 位置触发 Location | ◈ 短期 Short | 与重要记忆关联→◆ Linked to important memory → Long |
| 情绪触发 Emotion | ◈ 短期 Short | 情感强度>0.8→◆ Intensity >0.8 → Long |
| 记忆触发 Memory | ◆ 长期 Long | 用户反复阅读→★ User re-reads → Eternal |
| 深度对话 Deep Dialogue | ◆ 长期 Long | 涉及存在性主题→★ Existential themes → Eternal |
| 纪念日 Anniversary | ◆ 长期 Long | 用户标记珍藏→★ User cherishes → Eternal |

### 寿命标注位置

在每首诗的右下角，用符号标注：

```
2026.05.16 · 照片 · 晴天
                    ◈
```

## 生产流程 Production Flow

```
用户请求诗集 User requests collection
    ↓
系统自动收集诗歌（1-12个月的数据）Auto-collect poems (1-12 months data)
    ↓
自动筛选+用户确认 Auto-filter + user confirmation
    ↓
自动排版（模板化）Auto-layout (templated)
    ↓
用户预览+微调 User preview + adjustments
    ↓
送印（数字印刷，1本起印）Print (digital, 1 copy minimum)
    ↓
装帧+手工编号 Binding + hand numbering
    ↓
寄送 Ship
```

## 限量与编号 Limited Edition & Numbering

- 每本诗集都是唯一的（内容来自不同的用户）Each collection is unique (content from different users)
- 每本有唯一编号（如 #042）Each has unique number (e.g. #042)
- 不限量生产，但每本都是"限量1本" Unlimited production, but each is "limited to 1 copy"
- 如果同一用户多次制作诗集，编号连续（#042-1, #042-2）Same user's multiple collections get sequential numbers

---

> 实体诗集是数字陪伴的物理见证。它不需要精美，需要真诚。拿到手里的时候，用户应该觉得："这是我那段时间的诗。"
> The physical collection is a physical witness to digital companionship. It doesn't need to be exquisite, it needs to be sincere. When holding it, users should feel: "These are my poems from that time."
