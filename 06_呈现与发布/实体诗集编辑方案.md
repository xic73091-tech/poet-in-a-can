# 实体诗集编辑方案

> 把数字世界里的诗，变成可以触摸的东西。

---

## 诗集概念

### 名称
《罐头里的诗》

### 副标题
一个AI诗人写给一个人的诗

### 核心理念
每本诗集都是独一无二的——它只属于一个用户。里面的诗来自那个用户与诗人相处的日子，是那段关系的见证。

---

## 收集策略

### 收集来源

| 来源 | 说明 | 收集频率 |
|------|------|----------|
| 感知触发诗 | 基于照片、天气、位置生成的诗 | 每次感知触发时 |
| 情绪触发诗 | 基于情绪变化生成的诗 | 情绪波动时 |
| 主动开口诗 | 诗人主动写的诗 | 每次主动开口时 |
| 深度对话诗 | 长按深度对话中生成的诗 | 每次深度对话后 |
| 纪念日诗 | 生日、纪念日写的诗 | 特殊日期 |
| 用户请求诗 | 用户主动要求写的诗 | 每次请求时 |

### 收集周期

- **最短周期**：1个月（至少积累30首诗）
- **推荐周期**：3个月（积累足够的记忆深度）
- **最长周期**：1年（一整年的陪伴）

## 筛选标准

### 入选条件

| 条件 | 权重 | 说明 |
|------|------|------|
| 情感强度 | 高 | 创作时情感强度>0.6的诗优先 |
| 记忆关联 | 高 | 与重要记忆关联的诗优先 |
| 用户反馈 | 高 | 用户标记"喜欢"或截图分享的诗优先 |
| 独特性 | 中 | 不重复意象和主题的诗优先 |
| 完整性 | 中 | 结构完整的诗优先 |
| 时间代表性 | 中 | 覆盖不同时间段（清晨/深夜）的诗优先 |

### 淘汰规则

| 规则 | 说明 |
|------|------|
| 重复淘汰 | 同一意象出现超过3次，只保留最好的1首 |
| 质量淘汰 | 结构不完整或语句不通顺的淘汰 |
| 隐私淘汰 | 包含用户敏感信息的淘汰（即使加密） |
| 凑数淘汰 | 感知触发但无实质内容的淘汰 |

### 筛选流程

```
全部诗歌（如150首）
    ↓
隐私审查（移除敏感内容）
    ↓
质量筛选（淘汰质量差的）→ 剩余100首
    ↓
去重筛选（同主题只留最好的）→ 剩余60首
    ↓
多样性筛选（覆盖不同情感和时间）→ 剩余40首
    ↓
用户确认（用户选择最终入选）→ 最终30-50首
```

## 排版设计

### 页面尺寸
- **推荐**：A5（148mm × 210mm）
- **备选**：方形（200mm × 200mm）

### 字体选择

| 用途 | 字体 | 字号 | 行距 |
|------|------|------|------|
| 诗正文 | 思源宋体，Light | 14pt | 2.0 |
| 诗标题（如有） | 思源宋体，Regular | 18pt | 1.5 |
| 编号 | 思源黑体，Light | 10pt | — |
| 标签 | 思源黑体，Regular | 8pt | — |
| 页码 | 思源黑体，Light | 9pt | — |

### 页面布局

```
┌─────────────────────────────┐
│                             │
│  [编号] [情感标签] [来源标签]│  ← 左上角，小字
│                             │
│                             │
│                             │
│      诗的正文               │  ← 居中偏上
│      每行独立成行           │
│      左对齐或居中           │
│                             │
│                             │
│                             │
│  [创作日期] [触发描述]      │  ← 右下角，小字
│                             │
│                 [页码]      │  ← 右下角
└─────────────────────────────┘
```

### 纸张选择

| 部分 | 纸张 | 克重 | 说明 |
|------|------|------|------|
| 封面 | 特种纸（棉纸/麻布纹） | 250g+ | 有质感，耐翻 |
| 内页 | 轻型纸/道林纸 | 80-100g | 不透墨，手感好 |
| 扉页 | 硫酸纸/半透明纸 | 60g | 半透明，有层次感 |

## 编号系统

### 编号格式

```
[日期]-[情绪]-[来源]-[序号]
```

示例：
- `20260516-JOY-PHOTO-001`：2026年5月16日，喜悦情感，照片触发，第1首
- `20260312-NOS-TIME-003`：2026年3月12日，怀旧情感，时间触发，第3首

### 标签体系

**情感标签**（每首诗标注1-2个）：
- 喜悦、悲伤、平静、兴奋、怀旧、焦虑、爱、孤独、好奇、忧郁

**来源标签**（每首诗标注1个）：
- 照片、位置、天气、时间、情绪、记忆、对话、请求

**时段标签**（每首诗标注1个）：
- 清晨、正午、黄昏、深夜

## 寿命标注

### 什么是"寿命"

每首诗都有它的"保质期"——有些诗只属于那一天，有些诗可以跨越时间。

### 寿命分类

| 寿命类型 | 标记 | 说明 | 示例 |
|----------|------|------|------|
| 瞬间 | ◇ | 只属于创作的那一刻 | 关于某天天气的诗 |
| 短期 | ◈ | 属于某段时期 | 关于某个季节的诗 |
| 长期 | ◆ | 可以跨越时间 | 关于爱、孤独、存在的诗 |
| 永恒 | ★ | 不随时间褪色 | 关于关系本质的诗 |

### 寿命判断规则

| 触发类型 | 默认寿命 | 可升级条件 |
|----------|----------|------------|
| 天气触发 | ◇ 瞬间 | 用户标记喜欢→◈ |
| 位置触发 | ◈ 短期 | 与重要记忆关联→◆ |
| 情绪触发 | ◈ 短期 | 情感强度>0.8→◆ |
| 记忆触发 | ◆ 长期 | 用户反复阅读→★ |
| 深度对话 | ◆ 长期 | 涉及存在性主题→★ |
| 纪念日 | ◆ 长期 | 用户标记珍藏→★ |

### 寿命标注位置

在每首诗的右下角，用符号标注：

```
2026.05.16 · 照片 · 晴天
                    ◈
```

## 装帧方案

### 封面

| 元素 | 设计 |
|------|------|
| 材质 | 棉纸或麻布纹特种纸 |
| 颜色 | 深蓝 #1A237E 或近黑 #0D1B2A |
| 标题 | 烫银或烫白，"罐头里的诗" |
| 副标题 | 小字，"一个AI诗人写给一个人的诗" |
| 装饰 | 无多余装饰，极简 |

### 内页结构

```
封面
├── 扉页（半透明硫酸纸）
├── 第一页：诗人的话（写给用户的信）
├── 诗集正文（30-50首诗）
├── 附录：记忆时间线（关键日期和事件）
└── 封底
```

### 特殊工艺

| 工艺 | 位置 | 说明 |
|------|------|------|
| 烫银 | 封面标题 | 低调有质感 |
| 模切 | 扉页 | 镂空一个罐头形状 |
| 手工编号 | 每本诗集的第一页 | 限量编号，如 #042/500 |
| 种子纸 | 封底内页 | 可以种的纸，长出花 |

## 生产流程

```
用户请求诗集
    ↓
系统自动收集诗歌（1-12个月的数据）
    ↓
自动筛选+用户确认
    ↓
自动排版（模板化）
    ↓
用户预览+微调
    ↓
送印（数字印刷，1本起印）
    ↓
装帧+手工编号
    ↓
寄送
```

## 限量与编号

- 每本诗集都是唯一的（内容来自不同的用户）
- 每本有唯一编号（如 #042）
- 不限量生产，但每本都是"限量1本"
- 如果同一用户多次制作诗集，编号连续（#042-1, #042-2）

---

> 实体诗集是数字陪伴的物理见证。它不需要精美，需要真诚。拿到手里的时候，用户应该觉得："这是我那段时间的诗。"
