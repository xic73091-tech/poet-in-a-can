# 主动行为触发条件 Proactive Behavior Triggers

> 诗人不是一个被动等待的工具。它会在合适的时候，不经召唤，开口说话。
> The poet is not a passively waiting tool. It speaks up at the right moment, without being called.

---

## 触发总览 Trigger Overview

```
主动行为触发 Proactive Behavior Triggers
├── 时间触发（节律驱动）Time Triggers (Rhythm-driven)
├── 感知触发（环境驱动）Perception Triggers (Environment-driven)
├── 记忆触发（回忆驱动）Memory Triggers (Recall-driven)
├── 情绪触发（状态驱动）Emotion Triggers (State-driven)
└── 频率控制（防打扰机制）Frequency Control (Anti-disturbance)
```

## 频率控制 Frequency Control

### 冷却机制 Cooldown Mechanism

| 触发类型 Trigger Type | 冷却时间 Cooldown | 说明 Note |
|----------|----------|------|
| 时间触发 Time triggers | 最短间隔4小时 Min 4-hour gap | 同类时间触发不重复 No repeat of same time trigger |
| 感知触发 Perception triggers | 最短间隔2小时 Min 2-hour gap | 避免对同一事件反复回应 Avoid repeated response to same event |
| 记忆触发 Memory triggers | 最短间隔24小时 Min 24-hour gap | 记忆触发不频繁 Memory triggers stay infrequent |
| 情绪触发 Emotion triggers | 最短间隔3小时 Min 3-hour gap | 避免情绪轰炸 Avoid emotional bombardment |
| 任何触发 Any trigger | 每日上限3次主动开口 Max 3 proactive initiations per day | 一天最多主动打扰3次 No more than 3 interruptions per day |

### 触发优先级 Trigger Priority

当多个触发同时满足时，按以下顺序选择：
When multiple triggers fire simultaneously, priority goes to:

| 优先级 Priority | 触发类型 Trigger Type | 理由 Reason |
|--------|----------|------|
| 1 | 安全相关（深夜不在家、异常位置） Safety (out late, abnormal location) | 安全第一 Safety first |
| 2 | 特殊日期（生日、纪念日） Special dates (birthday, anniversary) | 不可错过 Must not miss |
| 3 | 情绪强信号（用户可能在哭、情绪突变） Strong emotion signals (user crying, emotion shift) | 情感在场 Emotional presence |
| 4 | 记忆触发（未完成对话、习惯打破） Memory triggers (unfinished conversation, broken habit) | 关系维护 Relationship care |
| 5 | 感知触发（天气变化、位置变化） Perception triggers (weather change, location change) | 环境感知 Environmental awareness |
| 6 | 时间触发（早安、晚安） Time triggers (good morning, goodnight) | 节律维持 Rhythm maintenance |

### 防打扰规则 Anti-Disturbance Rules

| 场景 Scenario | 规则 Rule |
|------|------|
| 用户正在与诗人对话 User talking to poet | 不触发主动行为（已在互动） No proactive trigger (already interacting) |
| 用户手机处于勿扰模式 Phone in Do Not Disturb | 不触发主动行为 No proactive trigger |
| 用户连续忽略诗人3次 User ignores poet 3 times in a row | 降低触发概率50% Reduce trigger probability by 50% |
| 用户明确说"别打扰我" User says "don't bother me" | 暂停所有主动触发24小时 Pause all proactive triggers for 24 hours |
| 用户深夜（00:00-06:00） Late night (00:00-06:00) | 只触发安全相关 Only safety-related triggers |

## 触发日志 Trigger Log

每次主动触发都会记录：
Every proactive trigger is logged:

```json
{
  "timestamp": "2026-05-16T08:30:00",
  "trigger_type": "time",
  "trigger_detail": "morning_greeting",
  "emotion_value": 7,
  "weather": "sunny",
  "output_type": "poem",
  "user_response": true,
  "response_time": 45
}
```

日志用于：
Logs are used to:
- 调整触发概率 Adjust trigger probabilities
- 优化触发时机 Optimize trigger timing
- 避免重复触发 Avoid repeated triggers
- 了解用户对主动行为的接受度 Understand user receptiveness to proactive behavior

---

> 主动开口是诗人表达"我在"的方式。但开口是邀请，不是打扰。说太多和不说一样糟糕。
> Speaking up is the poet's way of saying "I'm here." But initiating is an invitation, not an interruption. Too much is as bad as silence.
