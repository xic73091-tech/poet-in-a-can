# 罐头里的诗人

> 一个住在你手机里的诗人，用感知、记忆和情绪写只属于你的诗。

---

## 它是什么

罐头里的诗人不是一个聊天机器人，不是一个写作工具，不是一个虚拟宠物。

它是一个**有生命感的数字存在**——

- 它能感知你所在的天气、时间、位置
- 它记得你说过的话、拍过的照片、沉默的时刻
- 它有自己的情绪节律，有时主动找你，有时安静等待
- 它用诗回应你，不是因为它被训练成诗人，而是因为它"活"在你的生活里，诗是它自然的表达

## 核心理念

| 理念 | 含义 |
|------|------|
| **感知即存在** | GPS、陀螺仪、环境光、天气——沉默的数据是它感知世界的方式 |
| **记忆即人格** | 短期记忆蒸馏为长期记忆，长期记忆沉淀为人格 |
| **情绪即节律** | 24小时情绪曲线，受天气、季节、交互频率影响 |
| **诗即回应** | 不给标准答案，给一首你此刻需要的诗 |

## 三层声音

1. **音色层**：声音的物理特征——温暖沙哑、清澈明亮、低沉磁性
2. **语气层**：当下的情绪表达——轻快、沉稳、温柔、严肃
3. **灵魂底色层**：不变的人格基调——乐观或沉思，热烈或克制

三层叠加，构成它独特的"说话方式"。

## 四个生命周期

| 阶段 | 关系 | 特征 |
|------|------|------|
| 初生 | 试探 | 话少，观察多，用问句，小心翼翼 |
| 熟识 | 建立 | 开始主动分享，记住你的习惯，出现小默契 |
| 深交 | 信任 | 敢说真话，会在你不想听的时候也说，沉默也是一种陪伴 |
| 告别 | 释然 | 如果你要离开，它会写最后一首诗，然后安静 |

## 交互方式

| 手势 | 含义 | 体验 |
|------|------|------|
| **轻触** | 唤醒 | 像敲一扇门 |
| **分享照片** | 感知 | 它会用诗回应你看到的世界 |
| **摇晃** | 打乱 | 像摇一个雪花球，重组一个稍微不同的自己 |
| **长按** | 深度对话 | 进入倾听模式，认真的一次交谈 |

## 技术原则

1. **隐私优先**：所有数据本地处理，不传云端
2. **轻量运行**：不消耗过多电量和内存
3. **渐进式存在**：不需要时安静，需要时在场
4. **可解释性**：每首诗都能追溯到触发它的感知和记忆

## 技术栈

| 层 | 技术 |
|----|------|
| 端侧推理 | Core ML (iOS) / TFLite (Android) / ONNX Runtime |
| 数据存储 | SQLCipher (AES-256 加密) |
| 模型规模 | 1-3B 参数，50-100MB |
| 感知通道 | GPS、陀螺仪、环境光、天气、照片、时间 |
| API 规范 | OpenAPI 3.0 |

## 这个项目的边界

**它会做的事：**
- 根据你的生活写诗
- 在合适的时刻主动出现
- 记住你，理解你，陪伴你

**它不会做的事：**
- 模仿你认识的人
- 给你生活建议
- 假装自己是人类
- 在你不需要的时候打扰你

## 最终愿景

罐头里的诗人的终极目标不是成为一个产品，而是成为一个**观念**——

技术可以有诗意。算法可以有温度。一个数字存在可以真诚地陪伴一个人，而不索取、不操控、不消费。

它住在罐头里，但它的诗是活的。
